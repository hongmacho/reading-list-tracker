# PRD: BookMark - 책 읽기 추적 & 핵심 정리

## 1. 서비스 개요

**프로젝트명**: BookMark  
**영문명**: Reading List Tracker  
**슬로건**: 읽은 책, 핵심 인용, 개인 의견을 한곳에서 정리하다

### 문제 정의
사용자는 읽은 책을 기억하지 못하고, 핵심 내용을 재독할 때마다 처음부터 읽게 됩니다. 기존 앱(Goodreads)은 책 목록만 제공하거나 복잡하며, 단순 독서 추적에 불편합니다.

### 해결책
책 목록 관리 앱에서 읽는 진도(%)+ 핵심 인용 메모 + 개인 의견 기록을 통합 관리. 태그로 분류하고 검색 가능하게 함으로써 나중에 빠르게 복습 가능.

---

## 2. 타겟 페르소나

| 페르소나 | 특징 | 니즈 |
|---------|------|------|
| **학생/연구자** | 학기 중 다수의 도서 읽기, 학습 자료 재활용 필요 | 책별 핵심 인용 저장, 빠른 검색으로 과제 준비 시간 단축 |
| **전문가/직장인** | 업무 관련 서적 읽기, 배운 내용 실무 적용 | 직무별 태그 분류, 인용 공유, 읽기 진도 추적 |
| **독서 애호가** | 정기적 독서, 장르 다양화, 독서 습관 추적 | 월별 독서량 시각화, 장르별 분석, 별점 기반 추천 |
| **지식 정리자** | 읽은 책의 핵심만 정리, 나중에 참고 | 인용+메모+검색 통합, 극단적 단순함 선호 |

### 킬러 기능 (Must-Have 1순위)
**핵심 인용 메모와 개인 의견이 통합된 책 상세 페이지**: 책을 읽으며 직접 인용을 저장하고(최대 5개), 각 인용에 개인 의견을 덧붙일 수 있으며, 나중에 빠르게 검색/복습 가능. 이것이 Goodreads와 달리 BookMark만의 핵심 가치.

## 3. MoSCoW 우선순위 테이블

### **MUST-HAVE (8개): 반드시 구현해야 할 기능**

| # | 기능명 | 설명 | 서브기능 |
|----|--------|------|---------|
| 1 | 책 추가 & 관리 | 제목, 저자, ISBN, 카테고리 입력 | - 책 정보 자동 조회 (ISBN 기반)<br>- 책 이미지 업로드 |
| 2 | 읽는 진도 추적 | 현재 읽은 페이지/% 기록 | - 진도 변경 시간 기록<br>- 진도 그래프 시각화 |
| 3 | 핵심 인용 저장 | 책에서 건진 인용 저장 (최대 5개) | - 페이지 번호 포함<br>- 인용 태그 추가 |
| 4 | 개인 의견 메모 | 인용별 개인 해석/의견 (3줄 요약) | - 마크다운 지원<br>- 메모 날짜 기록 |
| 5 | 태그 기반 분류 & 검색 | 주제/카테고리/추천 등 태그로 분류 | - 제목 + 저자 + 인용 전문 검색<br>- 태그별 필터링 |
| 6 | 완료 표시 & 별점 | 책 읽기 완료, 1-5 별점 추가 | - 완료 날짜 기록<br>- 별점별 통계 |
| 7 | 월별 독서량 대시보드 | 월별 완독 권수, 진행 중인 책 현황 | - 장르별 읽은 책 분포<br>- 평균 읽기 시간 |
| 8 | 데이터 내보내기 | CSV 형식으로 모든 책 및 인용 내보내기 | - CSV 필터링 (기간별)<br>- JSON 형식 옵션 |
| 9 | 빈 상태 & 에러 처리 | 책이 없을 때, 네트워크 오류 시 메시지 | - 친화적 가이드 메시지<br>- 에러 재시도 UI |
| 10 | 도서 카테고리 자동 분류 | 소설, 비소설, 개발, 자기계발 등 | - 사용자 정의 카테고리<br>- 카테고리별 필터 (conditions 포함) |

## 4. 화면 목록 (6개 이상)

| # | 화면명 | 목적 | 주요 요소 |
|----|--------|------|----------|
| 1 | **대시보드** | 독서 현황 한눈에 보기 | 이달 읽은 권수, 진행 중인 책(최근 3개), 별점 분포 차트, "책 추가" CTA |
| 2 | **서재/책 목록** | 모든 책 열람, 필터/검색 | 책 카드(표지+제목+저자+진도%), 검색창(제목/저자/인용), 필터(완료/진행중/미시작, 카테고리, 별점) |
| 3 | **책 추가/편집** | 새 책 정보 입력 또는 수정 | ISBN 입력(자동 조회), 제목, 저자, 카테고리 선택, 표지 업로드, 저장/취소 버튼 |
| 4 | **책 상세 + 인용/메모** | 책의 모든 정보 열람 및 편집 | 책 표지+제목+저자+별점, 진도% 조정, 인용 리스트(최대 5개), 메모 추가/수정, 삭제 버튼 |
| 5 | **통계/분석** | 독서 패턴 시각화 | 월별 완독 권수(막대차트), 장르별 분포(원형차트), 평균 별점, 총 읽은 인용 개수 |
| 6 | **읽기 기록** | 진도 변경 이력 조회 | 날짜별 진도% 기록, 읽기 속도(page/day) 계산, 완료 예상일 |
| 7 | **설정** | 앱 설정 및 데이터 관리 | 카테고리 추가/수정/삭제, CSV 내보내기, 다크 모드 토글, 버전 정보 |

## 5. Conditions (아이디어의 조건사항) 반영

| Condition | 구현 방식 | 해당 화면/기능 |
|-----------|----------|--------------|
| 책 카테고리 자동 분류(소설·비소설·개발 등) | 사전 정의된 카테고리 8개 + 사용자 정의 | 책 추가/편집 화면, 설정 화면 |
| 핵심 인용은 최대 5개까지 저장 권장 | UI에서 5개 도달 시 "추가 버튼 비활성화" + 안내 메시지 | 책 상세 페이지 인용 섹션 |
| 검색: 제목·저자·인용 전문 검색 지원 | 통합 검색창 + 전체 텍스트 인덱싱 (SQLite FTS) | 서재 리스트, 검색 결과 페이지 |

## 6. 데이터 모델 (초안)

### Entity Relationship

```
books
├── id (PK)
├── title (string, not null)
├── author (string)
├── isbn (string, unique)
├── category (string) → categories (FK)
├── cover_image_url (string)
├── status (enum: 'not_started', 'reading', 'completed')
├── current_page (int)
├── total_pages (int)
├── rating (int, 1-5)
├── completed_date (timestamp)
├── created_at (timestamp)
├── updated_at (timestamp)

quotes
├── id (PK)
├── book_id (FK → books)
├── text (text, not null)
├── page_number (int)
├── tags (string, comma-separated)
├── created_at (timestamp)

notes
├── id (PK)
├── quote_id (FK → quotes)
├── content (text, markdown)
├── created_at (timestamp)
├── updated_at (timestamp)

book_tags
├── id (PK)
├── book_id (FK → books)
├── tag (string) → 주제/카테고리
├── created_at (timestamp)

reading_history
├── id (PK)
├── book_id (FK → books)
├── page_before (int)
├── page_after (int)
├── recorded_at (timestamp)

categories
├── id (PK)
├── name (string, unique)
├── icon (string)
├── is_default (boolean)
```

### 기본 카테고리 8개
1. 소설 (Fiction)
2. 비소설 (Non-Fiction)
3. 개발/IT (Technology)
4. 자기계발 (Self-Help)
5. 비즈니스 (Business)
6. 역사/인문 (History & Humanities)
7. 과학 (Science)
8. 기타 (Others)

## 7. 비기능 요구사항

| 요구사항 | 설명 | 기준 |
|---------|------|------|
| **성능** | 책 검색 응답 시간 | < 200ms (SQLite FTS 인덱싱) |
| **확장성** | 저장 가능 최대 책 | 5,000권 (로컬 SQLite) |
| **보안** | 데이터 암호화 | 로컬 저장소, 유출 위험 없음 |
| **가용성** | 오프라인 모드 | 100% 오프라인 지원 (클라우드 불필요) |
| **접근성** | 웹 접근성 | WCAG 2.1 AA 기준 준수 |
| **모바일** | 반응형 디자인 | 모바일(320px), 태블릿(768px), 데스크탑 |
| **브라우저** | 지원 범위 | Chrome/Safari/Firefox 최신 2버전 |
| **테스트 커버리지** | 최소 기준 | 80% (유닛 테스트 + 통합 테스트) |
